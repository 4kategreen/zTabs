Initialization

zTabs has to be initialized when the document is loaded, in order to use its tabbed interface. zTabs requires one unordered list with at least one tab, which will be used to build the first row of tabs.

zTabs has several configuration options which can be set at initialization to customize its interface. Options are specified by passing a standard javascript object into the .zTabs() method.

contentdivid and subrowsdivid:

One typical zTabs setup looks like this:

Example:
$(document).ready(function() {

	// mainTabs is id of the initial unordered list
	$('#mainTabs').zTabs({
		contentdivid: 'mainContent',
		subrowsdivid: 'subRows'
	});
});

Two common options have been specified above, contentdivid and subrowsdivid. By default, zTabs already manages tab content and additional rows of tabs. But for various reasons, such as styling zTabs using CSS, it can be useful to explicitly declare where content and subrows should go. When these options are set, zTabs will look for the divs specified by the values when placing content and rows.

localstorage:

zTabs uses HTML5 Local Storage by default to track tabs that have been opened. This allows zTabs to restore a previous browser session when returning to your site, or refreshing the page. Tab content is not saved via Local Storage. Internet Explorer versions prior to 8 do not support Local Storage.

In some cases, it may be useful to disable Local Storage for your zTabs installation. As an example, for sites which support individual user accounts, Local Storage might be disabled for users who have not yet logged in.

To disable Local Storage, set the localstorage option to 'ignore'.

Example:
$(document).ready(function() {
	$('#mainTabs').zTabs({
		...
		localstorage: 'ignore'
	});
});
