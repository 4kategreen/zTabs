HTML

zTabs is structured around HTML unordered lists. Each row of tabs is a ul. Each tab is a list item wrapped around an anchor tag.

zTabs requires one unordered list with at least one tab during initialization, for setting up its first row. When zTabs is initialized, the ul will be parsed and the tab interface will be built. After initialization, zTabs manages interactions with its interface through javascript, libraries like jQuery, and the DOM.

Unless dynamically generated by zTabs, every ul and li should be given a unique id.

Example:
// The ul used to initialize this zTabs installation looks something like this
<ul id='mainTabs'>
	<li id='tabOne'><a href="tabOne.html">Tab One</a></li>
	<li id='subTabs' data-closeable='true'><a href="pages/subtabs.html">Sub Tabs</a></li>
	<li id='demos'><a href="pages/demos/">Demos</a></li>
	<li id='docs'><a href="pages/docs/">Docs</a></li>
</ul>

Within a tab's list item, the anchor tag's target — the contentURL — should be one of two things:

  1. The tab's content.
  2. A subrow of tabs in the form of another unordered list.

When a tab is loaded for the first time, zTabs will examine the data, and determine if it is content or a subrow. If the first thing zTabs encounters is an unordered list — leading whitespace, which includes blank lines and spaces, is NOT permitted — the data will be treated as a subrow. In all other cases the data will be treated as tab content.

ContentURLs should be relative to the page which initializes zTabs.

By default, zTabs manages where it places subrows and content in the DOM. During initialization, zTabs can optionally be configured to place subrows and content into specific divs. This option may be useful when styling zTabs using CSS.

Example:
// For organization and styling, this zTabs installation uses 
// specific divs for subrows and content
<div id='tabNavigation'>
	<ul id='mainTabs'>
	...
	</ul>
	<div id='subRows'></div>
</div>
<div id='bodyWrapper'>
	<div id='mainContent'></div>
</div>
CSS

The look and feel of zTabs is highly customizeable using CSS.

Only a few styles are absolutely necessary for a functional zTabs interface.

Example:
// zTabs works, in part, by selectively showing and hiding tabs and content
.hiddenTab { display: none; }
.hiddenTabContent { display:none; }
		
// Tabs should usually be styled so that several tabs will fit on a row, by styling their widths
// The example below is just one of many ways to control tab widths
li { display: inline-block; }
		
// Current tabs should usually be differentiated from other tabs in some way
// The differences will usually be more extensive than in this example
li.current a, 
li.currentWithSecondRow a {
    color:red;
}
