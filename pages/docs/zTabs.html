.zTabs(<options>)

Builds a tabbed interface when pointed at an unordered list.

.zTabs() can be passed a number of options to configure the tabbed interface. Each option applies to either the collection of tabs itself ("tabset"), or to individual tabs. Options can later be changed using .zTabs('property').

This returns a jQuery object, so it is chainable.

Tabset Options:
contentdivid - Sets the div where content will be displayed for tabs. If this option is set, and there is any local content on the page, it should be placed in here as well.
subrowsdivid - Sets the div where rows of sub-tabs beyond the first row will be added. The first row must be set up as part of the original HTML. By default, sub-tabs are placed in the DOM after the set of tabs that contains their parent.
imagebaseurl - Sets the location where zTabs will look for a close button graphic. This defaults to 'library/images/'.
closebutton - The HTML for the close button. The default value is '<img width="11" height="11" border="0" src="library/images/close_tab_button_white.gif">'.
fromanchor - Sets which anchor links will automatically open in new tabs when clicked. The default value is 'a.zTab', so any anchor with class 'zTab' will cause a new tab to be created when it is clicked.
replace - Similar to 'fromanchor', this sets which anchor links will replace the current content with the results of an AJAX call to their href. The default value is 'a.zReplace', so any anchor with class 'zReplace' will cause the currently displayed content to be replaced when clicked.
formreplace - By default if you give a form the class 'zReplace', the form will be submitted via AJAX and the results will replace the current tab content. You can use the 'formreplace' property to set the class that's used for this feature to something else. The default value is 'form.zReplace'.
formresults - If you're using 'formreplace', you can set it up so the results from that AJAX call replace only a specific element by setting the element. The default is 'form.zReplace' but you can use this property to set it to anything you like.
Tab Options:
contenturl - Determines the source for a tab's content.
label - Sets the label for a tab.
maxlabelsize - This is the maximum number of characters that a label can have. Labels that are bigger than the maximum will be truncated to that size minus three, and "..." will be added. The full, un-truncated label will be visible via a tool-tip. This can be set globally when zTabs is initiated or it can be set specifically on an li. It can also be set for an anchor tag that has the class ztab or is processed by zTabsFromAnchor.
cache - Determines if a tab should go to the server and load its 'contenturl' every time the tab is clicked ('false') or just the first time ('true'). The default value is 'true'.
closeable - Determines if a tab is closeable. Closeable tabs typically feature a close button, which is configured using the 'imagebaseurl' and 'closebutton' options. Possible values for this option are 'true' and 'false'. The default value is 'false'.
onclose - A function to be called if the user presses the close button on a tab. If this function returns false then the close is cancelled.
oncreate - A function that gets called only the first time a tab is shown.
onsleep - A function that gets called every time a different tab is pressed and this is the active tab. If the onsleep function returns false then the switch to the new tab is cancelled.
onwake - A function to call every time the tab is shown.
singleton -
Examples:
$("#tabList").zTabs();

<ul id="tabList">
	<li><a href="#localContent">Local Content</a></li>
	<li><a href="pages/serverContent.html">Server Content</a></li>
	<li id='tabThree'><a href="pages/serverContent2.html">Server Content 2</a></li>
</ul>

As of version .9.9 JSONP can be used to grab a tab from a different domain. The content must be a JSON formatted object with the property html set to the content for the tab. That content can include a script tag which will be executed as it would if the content were coming from the same domain.

The href of the anchor tag should include callback=? in the URL past the query ? like so

		http://otherdomain.nrl.navy.mil/foo.php?bar=1&callback=?
